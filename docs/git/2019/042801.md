## 问题

遇到这么一个情况：分支A向分支B合并代码产生冲突，在线上解决后会自动将分支B上的代码反向合并到分支A，从而污染了分支A的代码。

## 处理

### 已经在线上解决

本地切换到分支A

```bash
# 将本地代码回滚到上一个版本
git reset --hard head^

# 强行将本地代码更新到线上
git push -f
```

### 没有在线上解决

本地切换到分支B

```bash
# 更新代码，将冲突拉到本地
git pull origin A

# 解决冲突，推到线上
```
